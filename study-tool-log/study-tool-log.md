# 학습 도구 로그

- 학습에 도움이 되는 도구를 만들고, 기록합니다. 필요하면 코드를 남깁니다.

## **도구 이름**

학습 전략 멘토

## **도구 유형** (예: GPTs, gems, Claude Code(skill, command, agent, ...) Chrome Extension, CLI, 웹사이트 등)

Gemini Gems

## **해결하려는 문제**: 어떤 학습 상의 불편/문제를 해결하려 했는가?

- 공부한 내용을 단순히 나열만 하고, 실제 내 것으로 만드는 리프렉션 과정이 부족함.
- 내가 무엇을 모르는지 모르는 상태에서 기록이 끝나는 문제.
- 작성한 학습 로그가 올바른 방향인지 스스로 판단하기 어려운 문제
- 로그를 남기는 행위 자체에 만족하고, 다음 단계의 의식적인 연습으로 이어지지 않는 현상.

## **어떻게 만들었는가**: 간단한 제작 과정

Gemini의 Gems 설정을 활용하여,  
학습 로그를 다양한 관점에서 점검하고 피드백해줄 수 있도록  
서로 보완적인 학습 이론을 가진 5인의 전문가 페르소나를 시스템 프롬프트로 결합했습니다.

이 도구의 목적은 로그를 작성하는 것 자체가 아니라,  
작성한 로그를 기반으로 학습 방향을 점검받는 것입니다.

- 에릭슨: 로그에 적힌 학습 내용 중 도전적인 부분이 빠졌는지 체크하고 교정 피드백 제공
- 오클리: 복잡한 학습 내용을 로그에 남길 때, 기억하기 쉬운 단위로 구조화하도록 유도
- 파인만: 로그 내용이 전문 용어에 의존하지 않고 실제 이해로 이어졌는지 질문으로 검증
- 스웰러: 한 로그에 과도한 정보를 담기보다 핵심 스키마 중심으로 정리하도록 가이드
- 스콧 영: 이 로그를 바탕으로 다음에 실행할 구체적인 행동(코드 작성 등)을 도출하도록 유도

## **어떻게 도움이 되었는가**: 실제 사용 경험과 효과

- 단순히 로그를 남기는 것이 아니라, 내가 올바른 방향으로 학습하고 있는지 지속적으로 점검할 수 있게 되었습니다.
- 아웃풋 중심 학습을 시도하면서 방향에 대한 의구심이 들 때마다, 이 Gems를 통해 지금은 완벽보다 실행이 우선이라는 기준을 다시 확인할 수 있었습니다.
- 5가지 학습 이론 기반 피드백을 통해 학습 상태를 객관적으로 진단받고, 확신을 가지고 학습에 몰입할 수 있었습니다.
