# 문제 #7

## 막혔던 부분

처음에는 v1과 v2의 타입을 User|Admin으로 했더니니 logUser과 logAdmin을 호출하는 부분에서 문제가 발생했다
예를 들면 v1이나 v2는 User일 수도 있고 Admin일 수도 있는데 logUser는 무조건 User을, logAdmin은 무조건 Admin타입을 원하고 있기 때문이다.

## 해결을 위해 필요한 키워드 1개

Tuple Type, 제너릭

내가 생각하는 키워드 : any
현재 swap 함수의 매개변수 에서 v1과 v2라는 매개변수의 타입을 추론할 수 없고, 명시적으로 선언되지도 않았을 때 나타나는 에러가 발생했다.

해결 방법으로는 매개변수에 명시적으로 타입을 지정해야 하지 않을까?
지금 타입을 알기 어렵기 때문에 any를 해야하지 않을까?

AI가 알려주는 키워드 : 제너릭
들어온 타입을 그대로 기억해서 내뱉어주는 제너릭이 필요하다

그럼 any는 안될까?
any를 쓰는 것은 타입스크립트를 안쓴다는 것과 같다.
그 변수에 대한 검사를 포기하는 행위이다.

## 키워드 단순 학습

제너릭은 함수의 타입을 매개변수로 사용하는 도구.

## 문제에 적용시켜 해결한 방법

제너릭 함수를 사용해서 들어오는 타입에 따라 반환되는 결과의 타입도 지정해준다.

## 한 줄 요약

타입을 좁히거나 제너릭을 통해 타입을 보존하는 연습이 중요하다!
제너릭을 사용해서 반환값 타입의 순서도 고정할 수 있다!

제네릭은 데이터의 정체성(타입)을 잃지 않게 지켜주는 보호막이고, 튜플은 그 데이터들이 서야 할 자리를 지정해 주는 이정표.
