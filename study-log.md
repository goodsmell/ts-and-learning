## 학습 로그 #1

**시간**: 03/24 16:00 ~ 16:30 (약 30분)

**학습 범위**: typescript-exercises #1~#3

### 1. 첫 학습 전략 설계

- 어떤 전략으로 학습하기로 했는가?

    일단 풀어보고, 이 문제에서 내가 얻을 수 있는게 무엇인지 알아본다.

- 그 전략을 선택한 이유는?

    어떤 것을 알아야 하고 중요한 것인지 인지하기 위해서이다.

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
    ```
    1. 문제를 푼다.
    2. 문제 풀기를 성공했을 경우, 해당 문제에서 집중해야할 키워드가 무엇인지 생각한다.
    3. 해당 키워드를 책이나 ai를 통해 학습한다.
    4. 다른 방식으로 풀 수 있는지 확인한다.
    ```

### 2. 전략 평가

- 효과적이었던 것과 그 이유

    해당 문제에서 내가 알아야 할 핵심, 키워드가 무엇인지 확인한 것.
    느낌으로 풀어내는 것이 아니라, 제대로 된 개념을 알 수 있었기 때문이다.

- 비효과적이었던 것과 그 이유

    너무 해당 키워드에 대해 깊게 파고들려고 한다.
    아는 내용에 대해서도 제대로 알고가려고 해서 시간이 너무 걸린다..

### 3. AI 피드백

- 자신의 학습 전략에 대해 AI에게 피드백을 요청하고, 유용했던 제안 1가지 이상 기록

> 너무 1부터 다 알려고 하는 습관을 바꾸기 위해 파인만 학습법을 역으로 사용해보라. "이 개념을 중학생에게 3분 안에 설명할 수 있는가?"를 기준으로 삼고, 만약 설명 가능하다면 그 이상의 탐구는 멈추고 다음 탐구로 넘어가라. 프로젝트를 하다가 막히면 그 때 더 깊은 탐구를 이어가라

> 이 개념을 내 프로젝트 어디에 적용해 볼 수 있을까? 라는 문장을 생각해봐라

> 현재 풀고 있는 문제와 직접 관련된 개념까지만 학습해라.

### 4. 다음 타임에 바꿀 것

- 유지할 것과 그 이유

    학습 전략은 유지한다. 타입스크립트 문제풀이에 효과적인 방법이라고 느껴진다

- 바꿀 것과 그 이유

    해당 키워드를 책이나 ai를 통해 학습할 때 현재 풀고있는 문제와 직접 관련된 개념까지만 학습하도록 하고, 중학생에게 설명할 수 있을 정도의 이해만 한다.
    그리고 각 문제에서 내 프로젝트 어디에 이런 것을 적용해 볼 수 있을까? 생각해본다.

    막히는 문제는 '이현'이 공유해준 템플릿을 적용해본다.

```
문제
막혔던 부분
해결을 위해 필요한 키워드 1개
키워드 단순 학습
문제에 적용시켜 해결한 방법
키워드 심화학습 (생각해 볼 만한 질문)
```

### 1. 이번 타임의 학습 전략

- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?

해당 키워드를 책이나 ai를 통해 학습할 때 현재 풀고있는 문제와 직접 관련된 개념까지만 학습하도록 하고 하였고, 문제마다 템플릿대로 생각 내용을 작성했다.

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.

1. 문제 풀기

< 안풀어질 경우 >

2. 책 목차를 보며, 해당 문제를 풀기 위해 필요할 것 같은 키워드 찾아보기
3. 키워드가 떠오르지 않으면 ai에게 키워드 묻기.
4. 키워드 공부하기
5. 문제 재시도

< 풀어질 경우 >

6. 각 문제 md파일에 내용 정리.

### 2. 전략 평가

- 효과적이었던 것과 그 이유

    각 문제 별로 md파일을 작성한 것이 효과적이 었던 것 같다.
    문제를 푸는 것에서 넘어가는 것이 아니라 그 문제에서 핵심이 무엇이고, 주요 개념이 무엇이고, 어떻게 풀어냈는지 정리가 되기 때문이다.

- 비효과적이었던 것과 그 이유

    각 문제에서 내 프로젝트 어디에 이런 것을 적용해 볼 수 있을까? 생각해보는 것이 이번 전략에 포함되어 있었는데, 아직 개념이 제대로 잡히지 않아 여기까지 생각하기에는 무리가 있었던 것 같다.

### 3. AI 피드백

- 자신의 학습 전략에 대해 AI에게 피드백을 요청하고, 유용했던 제안 1가지 이상 기록

1. ai에게 내가 찾은 키워드 외에 더 효율적인 typeScript 기능이 있는가? 물어보며 피드백의 밀도 높히기.
2. md 파일에 '나중에 이 개념이 헷갈릴 때 다시 볼 나만의 한 줄 요약'을 적어보며 뇌의 인출 속도를 높혀보자
3. 이 문법이 없었다면 javascript에서 어떤 런타임 에러가 났을까? 상상해보기.
4. md파일을 동료에게 설명하듯 구어체로 써보자!

### 4. 다음 타임에 바꿀 것

- 유지할 것과 그 이유

    **추론형 키워드 도출과 문제별 기록**

    문제가 막혔을 때 바로 답을 보지 않고 목차 → AI 키워드 질문 → 개념 공부 순으로 접근하는 방식이 머릿속에 더 각인시킬 수 있는 것 같다. 또한 .md 파일에 기록하는 작업을 통해 파편화된 정보를 하나로 묶어지는 것 같다.

- 바꿀 것과 그 이유

    **프로젝트 적용 고민을 해보기 -> 에러 시나리오 상상해보기**

    아직 개념이 생소한 상태에서 거창한 프로젝트 적용을 고민하는 것은 외재적 인지 부하만 높일 뿐이라는 ai 피드백을 받았다. 대신 '만약 이 타입이 없어서 순수 JS로 짰다면, 어떤 에러가 생겼을까?' 를 한 문장으로 적어보는 것이 더 효과적이다.
